Linking single-period attribution effects.

Single-period Brinson allocation and selection effects don't sum across multi-period analyses. The Frongello (2002) linking algorithm fixes that without redistributing a residual — and it preserves the order-dependence that the underlying portfolio mechanics demand. This page contains everything you need to understand it, compute it, ship it.

§ 02 — Formula

The formula, and what each piece means.

Reproduced verbatim from Frongello's original paper. Two terms. One scales the current effect by everything that came before. The other lets the cumulative prior effects compound at the current benchmark return.

— The Frongello adjustment —
A′t  =  At × ∏i=1t−1(1 + Rpi)  +  Rbt × ∑i=1t−1 A′i

Symbols.

SymbolMeaning
A′tAdjusted attribution effect at period t
AtRaw single-period attribution effect at t
RpiPortfolio return at period i
RbtBenchmark return at period t
nNumber of periods

Boundary conditions.

At t = 1, the empty product equals 1 and the empty sum equals 0:

A′₁ = A₁ × 1 + Rᵦ₁ × 0
   = A₁

The first period's adjusted effect always equals its raw effect. Use this as your first sanity check on any implementation.

The recursion, in code.

The natural implementation is a single forward pass with two running accumulators. Updates happen at the end of each iteration — that's the only ordering that satisfies the t−1 bounds.

def frongello_smooth(A, R_p, R_b):
    # A: raw effects per period
    # R_p, R_b: portfolio and benchmark returns
    cum_growth = 1.0
    cum_adj    = 0.0
    out = []
    for t in range(len(A)):
        a_prime = A[t] * cum_growth + R_b[t] * cum_adj
        out.append(a_prime)
        cum_growth *= (1 + R_p[t])
        cum_adj    += a_prime
    return out

The identity that matters.

The whole point of a linking algorithm is that the adjusted effects must sum exactly to the cumulative active return:

∑ A′t  =  ∏(1+Rpt) − ∏(1+Rbt)

Residual greater than 1e-9? You have a bug. The calculator above checks this on every keystroke.

Reverse the period order and you'll get different per-period values — but the same total. That's the order-dependence property: a feature of Frongello that distinguishes it from Carino and Menchero, both of which are order-independent.
